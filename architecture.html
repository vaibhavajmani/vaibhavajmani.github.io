A sales data pipeline that runs itself

vAI-RetailCo runs 340 stores. Every night, their point-of-sale SaaS exports inventory adjustments as a CSV. The central inventory API needs to be in sync before morning replenishment runs, or reorder quantities go out wrong. This walkthrough shows the design, the safeguards, and the reasoning behind each choice.

vAI-RetailCo is a fictional company. The architecture reflects a common enterprise integration pattern.
